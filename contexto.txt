# Prompt para desarrollar la aplicación

Quiero desarrollar una aplicación robusta y escalable para monitoreo y administración de servidores donde se ejecuta Odoo 9 con python 2.7.

## Objetivo general

La aplicación debe permitir configurar múltiples servidores y monitorear en tiempo real el estado, rendimiento y salud de cada uno, incluyendo métricas del sistema operativo, procesos de Odoo y estado de la base de datos PostgreSQL.

La solución debe enfocarse en:

* Alto rendimiento.
* Bajo consumo de recursos.
* Escalabilidad.
* Seguridad.
* Fácil despliegue.
* Buenas prácticas de desarrollo.
* Arquitectura mantenible.

---

# Funcionalidades principales

## 1. Autenticación y usuarios

La aplicación debe incluir:

* Login seguro.
* Manejo de sesiones.
* JWT o autenticación moderna.
* Roles y permisos.
* Recuperación de contraseña.
* Auditoría básica de accesos.

Roles sugeridos:

* Administrador.
* Operador.
* Solo lectura.

---

# 2. Gestión de servidores

Debe permitir:

* Registrar múltiples servidores.
* Editar/eliminar servidores.
* Configurar:

  * IP o dominio.
  * Puerto.
  * Usuario SSH.
  * Clave o llave SSH.
  * Tipo de servidor.
  * Ambiente (producción, pruebas, desarrollo).
  * Intervalos de monitoreo configurables por servidor.

La aplicación debe validar conectividad antes de guardar la configuración.

---

# 3. Monitoreo del servidor

La aplicación debe monitorear:

## Métricas del sistema

* Uso de CPU.
* Uso de RAM.
* Uso de SWAP.
* Espacio en disco.
* IO Disk.
* Temperatura (si aplica).
* Load average.
* Uso de red.
* Procesos activos.
* Servicios caídos.
* Reinicios del servidor.

## Métricas de Odoo

* Procesos de Odoo activos.
* Workers activos.
* Procesos colgados.
* Consumo de memoria de Odoo.
* Consumo de CPU por proceso.
* Tiempo de respuesta.
* Logs críticos.
* Cantidad de peticiones concurrentes.
* Estado de cron jobs.

## Métricas PostgreSQL

* Conexiones activas.
* Queries lentas.
* Bloqueos.
* Deadlocks.
* Procesos colgados.
* Tiempo de ejecución de consultas.
* Uso de índices.
* Tamaño de tablas.
* Replicación (si aplica).
* Estado general de la base de datos.

---

# 4. Sistema de alertas

Debe incluir:

* Alertas configurables.
* Umbrales personalizados.
* Notificaciones por:

  * Email.
  * Telegram.
  * WhatsApp (opcional).
  * Webhooks.
* Alertas automáticas cuando:

  * El servidor esté caído.
  * Odoo no responda.
  * PostgreSQL tenga bloqueos.
  * CPU/RAM exceda límites.
  * Espacio en disco esté crítico.

Las alertas deben evitar spam mediante cooldowns configurables.

# 9. Instalación y despliegue

Quiero que la instalación sea extremadamente simple.

La idea es:

* Clonar repositorio.
* Configurar `.env`.
* Ejecutar Docker Compose.
* Que todo se instale automáticamente.

Debe incluir:

* Dockerfile optimizado.
* docker-compose.yml.
* Scripts de inicialización.
* Migraciones automáticas.
* Seeds básicos.

Idealmente:

* Un solo comando para levantar todo.

Ejemplo:

```bash
docker compose up -d
```
